💡 Two Ways to Analyze
Manual Entry — type a single transaction's details and get instant risk prediction from your Flask model.

File Upload — upload a CSV, Excel, or PDF bank statement. The system extracts every transaction and analyzes them all with a live progress bar.
📂 Supported Formats
CSV / XLSX — map your Amount and Description columns, click Analyze All.

PDF — bank statement PDFs (SBI, HDFC, ICICI, Axis etc.) are parsed automatically by extracting ₹ amounts and row text.

Results export as CSV with risk scores for every transaction.
📊 Input Guide
Amount — transaction value in INR (₹)

Time — seconds since first transaction in the dataset (0–172792)

Anomaly Score — proxy for PCA features V3–V28. Higher values (8–10) simulate abnormal behavior that the model may flag as suspicious.

V1 / V2 — direct PCA feature overrides. Typical fraud values: V1 < -3, V2 > 1.5
⚠ Risk Thresholds
GENUINE Fraud probability < suspicious threshold
SUSPICIOUS Near-threshold — manual review
FRAUD Fraud probability ≥ main threshold
🔌 Backend Status
Make sure python app.py is running on localhost:5000 before analyzing. If the server is offline you'll see a connection error in the result box. Results are automatically saved to transactions.db and appear in the History tab.
